# 發票快掃核對精靈 (OCR_Module_Design)

## 1. 模組簡介
本模組為「應付科目餘額小幫手」的擴充套件。旨在解決會計人員處理實體發票/收據時，與 ERP 內帳核對耗時且實體歸檔容易混亂的痛點。
系統透過「電腦端架設本地伺服器」與「手機端相機拍照」的跨裝置協作，達成**完全離線（無資料外流）**的自動化 OCR 辨識、帳務比對與實體排列指引。

## 2. 系統架構 (跨裝置 LAN 同步)
為了達成「零網際網路需求」且流暢的體驗，架構設計如下：
- **Host (電腦端 Electron Main Process):**
  - 內建 `Express.js` 啟動一個輕量級 Local Web Server (例如 `http://<電腦內部IP>:3000`)。
  - 內建 `tesseract.js` (離線 OCR 引擎)，負責接收手機傳來的圖片並進行文字解析與特徵擷取 (金額、稅額、統編)。
  - 透過 IPC (Inter-Process Communication) 與 Angular 前端溝通。
- **Client (電腦端 Angular Frontend):**
  - 顯示 ERP 帳務清單。
  - 產生並顯示 LAN IP 與 QR Code 供手機掃描。
  - 透過 IPC 監聽後端傳來的「最新比對結果」，即時更新 UI (例如將某筆帳標示為✅已核對)。
- **Mobile (手機端 Browser / PWA):**
  - 會計手機連接公司 Wi-Fi，掃描 QR Code 進入本地網頁。
  - 提供單純的「相機拍照/相簿上傳」介面。
  - 圖片透過 `POST` 傳回電腦端。
  - 接收電腦端回傳的比對結果與「實體放置位置 (Grid Position)」，並提供錯誤時的手動輸入介面。

## 3. 核心演算法
### 3.1 OCR 處理策略
- 使用 Tesseract.js 載入 `chi_tra` (繁中), `chi_sim` (簡中), `eng` (英文數字) 模型。
- **正則表達式萃取 (Regex Extraction):** 針對 OCR 產出的純文字，使用正規表示式抓取：
  - 總金額：尋找 `總計|合計|Amount|Total` 附近的數字。
  - 稅額：尋找 `稅額|Tax|營業稅` 附近的數字。
  - 發票號碼：尋找符合 `[A-Z]{2}-\d{8}` 或類似格式的字串。
- **手寫容錯機制：** 若 OCR 失敗或抓不到金額，回傳 `needs_manual_input`，手機端彈出快速輸入框，會計手敲金額後重新送出比對。

### 3.2 帳務比對與定位邏輯 (Grid Positioning)
- 系統讀取先前匯入並完成 Mapping 的 `Vouchers` (憑證清單)，視為「預期帳單」。
- 當收到 OCR 萃取的金額 `X` 時，尋找 Vouchers 中 `credit - debit == X` 且尚未核對的單據。
- **成功匹配 (Matched):**
  - 假設該單據在清單中是第 `N` 筆 (1-based)。
  - 採用 10x10 網格邏輯：
    - 列 (Row) = `Math.ceil(N / 10)`
    - 張 (Col) = `N % 10` (若為 0 則是第 10 張)
  - 回傳結果：`✅ 匹配成功！請放置於：第 {Row} 列 / 第 {Col} 張`。
- **找不到匹配 (Unmatched):**
  - 若 ERP 帳上沒有符合該金額的紀錄，回傳：`⚠️ 找不到此帳款！`。
  - 手機端指示：`請放置於：【異常區 / 待確認盒】`。

## 4. 安全性與隱私
- **Local Only:** 所有圖片傳輸僅在區域網路內 (LAN) 發生。
- **No Storage:** 圖片在伺服器記憶體中處理完畢（或暫存於系統 Temp 內完成 OCR）後即刻銷毀，不留下實體圖片檔，杜絕商業機密外流。
