# 系統分析文件 (System Analysis - SA)

## 1. 系統簡介
本系統專為「會計人員（應付職務）」量身打造，旨在提供一套 **本地端運行、無資料外流風險、高效率** 的科目餘額彙整與檢查工具。系統著重於從原始資料（Excel/CSV）快速整理出科目餘額表，並自動稽核帳務異常。

## 2. 系統目標
- **絕對安全：** 系統無狀態（Stateless），資料僅在記憶體中處理，關閉程式即銷毀，杜絕資料外洩。
- **高效率：** 透過匯入檔案與動態欄位對應，取代繁瑣的人工輸入。
- **防錯稽核：** 內建多種會計異常檢查規則，自動揪出問題帳務。
- **多幣別支援：** 自動處理多幣別混雜情況，並支援匯率轉換。

## 3. 使用者角色
- **主要使用者：** 會計人員（負責應付帳款）。
- **使用情境：** 取得 ERP 匯出或廠商提供的交易明細（CSV/Excel）後，匯入本系統快速檢查異常，並產出最終的正確科目餘額表。

## 4. 核心功能規格
### 4.1 檔案匯入與動態對應 (Dynamic Mapping)
- 支援 CSV, Excel (XLSX) 格式。
- 系統自動讀取檔案表頭，使用者可透過下拉選單將檔案欄位對應至系統所需的核心欄位：
  - 憑證日期 (Date)
  - 憑證號碼 (Voucher No)
  - 會計科目 (Account Code / Name)
  - 供應商 (Vendor)
  - 借方金額 (Debit)
  - 貸方金額 (Credit)
  - 幣別 (Currency)
  - 匯率 (Exchange Rate)

### 4.2 異常防錯引擎 (Anomaly Detection)
系統匯入後自動執行以下稽核：
1. **借貸不平：** 單張憑證借貸金額不相等。
2. **餘額異常：** 應付帳款等負債科目出現「借方餘額」（可能重複沖銷或溢付）。
3. **資料缺失：** 缺乏供應商、會計科目或金額資料。
4. **匯率異常：** 同幣別在相近期間匯率落差過大（警示）。

### 4.3 多幣別處理與科餘表產生
- 系統將各幣別獨立計算餘額。
- 提供統一口徑的「本位幣（如新台幣）」折算視角。
- 計算公式： `期末餘額 = 貸方總計 - 借方總計`（針對應付科目）。

### 4.4 報表匯出
- 將整理好的「無異常」科目餘額表匯出為 CSV/Excel，方便會計後續歸檔或上報。

## 5. 非功能性需求
- **介面設計：** 乾淨簡潔、圓潤風格（Rounded UI），減少視覺疲勞，醒目標示錯誤。
- **效能：** 能快速解析至少一萬筆級別的資料列，不卡頓。
